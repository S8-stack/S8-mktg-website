core.bohr

BOHR

The Ubiquitous data format

BOHR (initially Binary Object for HTTP Request) is binary stream format that makes mulitpart, zipping, versioning and much more seamless integrated. BOHR protocol can be used for:

  • Persistency: for storage of complex Graph objects, including the serialization of many data structure.
  • Stream to Front: This is the key to send compelx structure to front with extremely tight data frame: every single object sent to front is part of graph wehere every node is identified. Therefore, any modification of the screen does not need any reload of previously transmitted objects.
  • Native Undo: Since all objects are persisted in a flow of sequential modifications, undo reconstruction comes at no costs.

    S8-core-arch-silicon

    The Heartbeat of the system

    The S8 core system is embedding a powerful application-wide thread pool workers with Task profile identification. Everything is packed under the SiliconEngine object and is available everywhere.

    Support different types of tasks

    S8 core arch silicon supports different types of tasks, each tailored for specific cases:

    • Reasonnably non-blocking tasks: (see: AsyncSiTask) Those tasks are expected to end in a finite amount of time. In order to have a reasonnable orchestration, each of thme come with a profile that indicates the expected run duration.
    • Blocking tasks: (see: WatchSiTask) Those tasks can block indifinitely and are therefore parked in a specific area of the engine.
    • Timed tasks: (see: CLockSiTask) Those tasks are periodically called back.

    S8-core-bohr-atom

    The Reference serialization protocol for all things S8

    Bohr is a kind of ultimate protocol that tremendously upgrades the currently ubiquitous JSON format. Its super powers come for the fact that it natively enables:

    • Efficient Serialization: serialize pretty much everything and natively support objects that come with a specific native serilization methods (blending), like for instances matrices, vectors, pictures.
    • Native Compression: BOHR is binary protocol, enabling bit-omptimized flow for file size and streams bandwidth maximal compression. Virtually all repetitions are removed in a single pass, enabling a merge of the serialization pass and the compression pass into an ultimately fast processing.
    • Self-standing: BOHR is an auto-documented protocol, meaning that all structures of objects are completely recorded into the file/stream, in a incremental way tolreant to data model evolutions. This enables to re-use any data processed through BOHR in anay application that support the BOHR protocol, allowing data-model reconstruction direcly from data themselves in an univoque manner.
    • Sequence of Deltas: Think of BOHR as a block chain type protocol. This implies that data can always be reconstructed through time, and that any state in time is always accessible through re-redaing the sequence of updates. This makes the do/undo pattern very easy to implement. Also, in the event of partial data destruction, reconstruction methods can still be used to recover a lot of things.

    Usages

    BOHR has the following usages for the S8 Stack:

    • Persistency: use for the Beryllium, Neodymium, and Lithium objects peristency handlers.
    • Streaming: BOHR is massively used to push objects from the back to the front, leveraging the native caching it offers.
